3. 交互行为总结

部署阶段 (DEPLOYMENT)

  • Grid 居中，scale = 1.0
  • 拖拽平移：禁用
  • 底部部署 UI：可见
  • BattlefieldContainer position 复位到初始位置

战斗阶段 (RUNNING)

  • Tween 缩放至 0.8~0.9（可调）
  • 缩放锚点：Grid 中心
  • 拖拽平移：启用，范围限制在战场边界内
  • 底部部署 UI：隐藏（Dev 模式除外）
  • 敌人在距 Grid 3 Cell 处生成
  • Dead Zone 在战场边缘

4. 状态转换时序

── 点击"开始战斗" ──────────────────────
1. 隐藏部署 UI（非 Dev 模式）
2. Tween: BattlefieldContainer.scale → 0.85, 约 0.3s
3. 启用拖拽平移输入
4. 创建 Dead Zone（战场边缘）
5. 生成敌人（距 Grid 3 Cell）
6. 塔开始射击

── 战斗结束（胜利/失败） ────────────────
1. 停止射击、清除子弹、移除敌人
2. 移除 Dead Zone
3. 禁用拖拽平移
4. Tween: BattlefieldContainer.scale → 1.0, 约 0.3s
5. Tween: BattlefieldContainer.position → 初始位置, 约 0.3s
6. 显示部署 UI

需要修改的文件

  • main.tscn — 新增 BattlefieldContainer Node2D，GridRoot 移入其下
  • 新增 core/battlefield_container.gd — 拖拽平移 + 缩放逻辑
  • main.gd — 初始化 BattlefieldContainer，战斗时隐藏/显示部署 UI
  • core/GameLoopManager.gd — 触发缩放/平移启停
  • core/dead_zone_manager.gd — Dead Zone 定位改为基于战场范围
  • entities/enemies/enemy_manager.gd — 生成距离改为 3 Cell
  • autoload/SignalBus.gd — 可能新增 battlefield 相关信号
