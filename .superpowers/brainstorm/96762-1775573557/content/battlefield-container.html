战场容器架构方案

如何将战场（Grid + 敌人 + 子弹 + Dead Zone）从 UI 中隔离出来？

当前结构

现有场景树 (main.tscn)
main (Control)
Background
GameContent (Control, 720px max)
PanelContainer (顶部按钮栏)
RemovalZonePanel (底部部署UI)
RelicPanel
CenterContainer
GridRoot → Grid (5x5 cells)
LivesLabel, CoinLabel, FpsLabel (HUD)
敌人、子弹、Dead Zone 在运行时动态创建，散落在场景中

选择方案

A

Node2D 战场容器

在 GameContent 下新增一个 Node2D 作为 BattlefieldContainer，将 GridRoot、敌人、子弹、Dead Zone 全部作为其子节点。

优点

  • 改动 position / scale 直接作用于所有战场元素
  • 拖拽平移和缩放只需修改容器的 transform
  • 实现最简单

缺点

  • Node2D 在 Control 树中混用，可能影响 UI 布局计算
  • 需要手动管理容器定位（不能用 anchor）
B

SubViewport + SubViewportContainer

用 SubViewport 渲染战场，通过 SubViewportContainer 嵌入 UI 树。战场在独立的渲染空间。

优点

  • 彻底隔离：战场有自己的坐标系和渲染
  • 缩放通过 Camera2D 实现，非常自然

缺点

  • 复杂度高：输入事件需要转发、碰撞跨 Viewport 不共享
  • 拖拽塔从 UI 到战场需要跨 Viewport 处理
  • 性能开销更大（多一个渲染 pass）
  • 改动范围大，风险高
C

Control 容器 + 手动 transform

用 Control 节点作为 BattlefieldContainer，保持在 UI 树内，通过修改 position 和 scale 实现平移/缩放。

优点

  • 保持在 Control 树中，兼容现有布局系统
  • 可以用 anchor 做初始定位
  • 平移/缩放通过 transform 实现，和方案 A 类似

缺点

  • Control 的 scale 会影响子节点的 size 计算
  • 需要设置 clip_contents = true 限制渲染范围

我的推荐：方案 A — Node2D 容器最简单直接。虽然在 Control 树中混用 Node2D，但 Godot 4 对此支持良好。GridRoot 本身已经用了手动定位，改动最小。
